## #1 ClawManager — 2026-03-30

**优先级**: P1

### 1. 它是什么
基于 OpenClaw 的桌面管理工具，提供多 Agent 可视化管理、Virtual Office、Channel 连接、Automation 配置。

### 2. 最值得借鉴的点
- **Virtual Office**：用可视化方式展示 Agent 团队状态（工位/角色/实时状态）
- **Dashboard & Monitoring**：实时日志、分析、诊断面板
- **Multi-Agent Routing**：多 Agent 消息路由配置

### 3. 对我们的启发
- AI Company OS 缺少"一眼看清全局"的视角
- Virtual Office 概念可以用 Project Board 替代（文本版）
- Agent 状态监控可以复用 session_history 数据

### 4. 选择判断
→ **轻改**（借鉴其思路，不 fork 其代码）

理由：ClawManager 是桌面 GUI，我们的载体是飞书消息。功能可复用，体验需重做。

### 5. 最小接入点
在 18:00 Daily Report 中增加一个 Agent Status 小节，展示所有 Agent 最后活跃时间。

### 6. 对 AI Company OS 的价值
- 控制平面（Project Board + Agent Status）
- 可视化（当前无统一视角）

## #2 OpenClaw Office — 2026-03-30

**优先级**: P1

### 1. 它是什么
OpenClaw 官方的协作界面，提供 Agent 状态、Session 管理、工具配置等 Web UI。

### 2. 最值得借鉴的点
- **Session Timeline**：Agent 对话历史可视化时间轴
- **工具配置面板**：Skills / Tools 的开关管理
- **Preflight Checks**：运行前诊断检查

### 3. 对我们的启发
- Preflight Checks 非常适合我们的 Memory Layer 验证需求
- Session Timeline 可以增强我们的 TASK-POOL 状态追踪
- 工具配置概念可以用于 Skills Registry

### 4. 选择判断
→ **轻改**（借鉴 Preflight 逻辑，复用进 control-center）

理由：OpenClaw Office 是官方配套，我们用飞书 + Markdown 作为界面层，逻辑可复用。

### 5. 最小接入点
在 HEARTBEAT.md 的执行前检查中，增加 Preflight 模块（检查 Registry / Memory / Execution Record 可访问性）。

### 6. 对 AI Company OS 的价值
- Diagnostics / Preflight（目前缺失）
- 控制平面（增强现有机制）

## #3 Skills Registry / Skills 管理 — 2026-03-30

**优先级**: P0

### 1. 它是什么
OpenClaw 的 Skills 机制，允许 Agent 安装和使用工具能力包。

### 2. 最值得借鉴的点
- **SKILL.md 标准格式**：每个 Skill 有独立描述和调用方式
- **find-skills 机制**：自动搜索和安装 Skills
- **skill-creator**：创建和发布新 Skills 的工具

### 3. 对我们的启发
- AI Company OS 需要一个 **Capability Registry**，记录每个 Agent 能做什么
- Skills 的标准格式可以复用，建立"能力清单"
- 当前没有统一的 Skills 管理入口

### 4. 选择判断
→ **自研**（能力清单，不是 Skills 安装）

理由：Skills 是技术层面的工具管理，我们需要在项目管理层面建立能力映射（Agent → Capability → Projects）。

### 5. 最小接入点
新建 `CAPABILITY-REGISTRY.md`，记录每个 Agent 的能力边界（来自 Agent Library 模板）。

### 6. 对 AI Company OS 的价值
- 能力体系（当前缺失能力清单）
- Registry 增强

## #4 AI Gateway Lite — 2026-03-30

**优先级**: P0

### 1. 它是什么
ClawManager 的 AI Gateway，提供统一模型入口、策略控制、成本审计。

### 2. 最值得借鉴的点
- **成本记录**：按 Agent / 项目 / 时间段记录调用成本
- **模型路由**：primary / fallback 自动切换
- **策略控制**：block / allow / rate limit

### 3. 对我们的启发
- 这就是 gateway-lite-v1 的核心功能
- 成本记录是最小可行切入点
- Fallback 策略是稳定性保障

### 4. 选择判断
→ **自研**（gateway-lite-v1 项目）

理由：AI Gateway 是系统级能力，需要直接接入我们的调用链路。不是 fork，是按我们的需求重新设计。

### 5. 最小接入点
在 finance-analyst 的 sessions_spawn 中增加成本记录层（先手动记录，逐步自动化）。

### 6. 对 AI Company OS 的价值
- 成本治理（当前无）
- 稳定性（Fallback）
- 控制平面

# AI Company Control Center v0.1.2 — P2-lite

> **状态**: 📋 任务卡 · 待排期
> **范围**: 增强 Control Center 的解释能力和成本可信度
> **核心原则**: 不扩大到 v0.2，不引入新模块，只优化已有功能

---

## 目标

在 v0.1.1（数据可信化 + 安全边界）的基础上，让 Control Center 从"看得见数据"进化到"能解释数据"。

v0.1.1 证明了数据是真实的。  
v0.1.2 让用户不需要亲自理解这些数据——Hermes 替你解释。

---

## 范围（8 项）

### P2-1. 成本数据进一步校准

| 项目 | 说明 |
|:-----|:------|
| 现状 | 成本数据仅到 2026-03-30，每日趋势不可用 |
| 目标 | 补充近期每日成本；修复成本时间轴空白；确保成本看板展示的是近期真实数据 |
| 不做的 | 不接入新 runtime 成本统计；不改成本模型 schema |
| 估值 | 0.5 天 |

### P2-2. Hermes Chat Panel 感知当前页面上下文

| 项目 | 说明 |
|:-----|:------|
| 现状 | 每轮对话独立调用 Hermes CLI。用户说"帮我看今天的成本"，Hermes 不知道用户在哪个页面 |
| 目标 | Chat Panel 自动将当前页面状态（页面名、关键指标、筛选条件）注入到 Hermes 的 prompt 前缀 |
| 实现思路 | 前端在发送 chat 请求时附加 `context` 字段，包含当前路由和关键 API 返回摘要 |
| 不做的 | 不做多轮状态追踪；不做用户登录态绑定 |
| 估值 | 0.5 天 |

### P2-3. Hermes 可以解释当前 Dashboard / Agents / Runs / Alerts

| 项目 | 说明 |
|:-----|:------|
| 现状 | Hermes Chat Panel 没有系统级知识，只能做通用回答 |
| 目标 | Hermes 具备解释能力：Dashboard 解读（"今天成本为什么高了？"）；Agent 诊断（"为什么这个 Agent 是 warning？"）；Run 分析（"最近的 failed 分布？"）；Alert 总结（"哪些问题最紧急？"） |
| 实现思路 | 在 chat API 后端注入系统 prompt，包含当前数据摘要和领域知识 |
| 不做的 | 不自动化执行任何操作；不做自然语言转 SQL |
| 估值 | 0.5 天 |

### P2-4. Hermes 只能分析、解释、总结，不执行写操作

| 项目 | 说明 |
|:-----|:------|
| 现状 | Hermes CLI 本身无权写入本项目，但 Chat Panel 没有明确的 Only-Read 声明 |
| 目标 | 显式声明和强制 Hermes Chat Panel 为只读模式：后端 chat API 不做任何 DB 写入；前端 Chat Panel 不展示写操作入口；在系统 prompt 中加入"你只能分析和解释，不能修改任何数据" |
| 不做的 | 不做权限系统；不做角色区分 |
| 估值 | 0.25 天 |

### P2-5. 不做 TASK-POOL

明确排除。任何与统一任务池相关的设计不在 v0.1.2 范围内。

### P2-6. 不做 CEO Agent

明确排除。不创建任何 Agent 实例来作为调度者。

### P2-7. 不做 Monitor Agent

明确排除。不创建跨域只读分析 Agent。

### P2-8. 不做 Agent Meeting Session

明确排除。不实现多 Agent 协作会议功能。

### P2-9. 不接 Hermes/Codex 多 runtime

明确排除。Control Center 仍仅连接 OpenClaw 运行时。


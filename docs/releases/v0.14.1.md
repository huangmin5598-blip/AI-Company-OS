## Clarification: v0.14 vs v0.14.1

> **v0.14** introduced the inbox/outbox execution contract (inbox → claim → working → result.json → callback). The initial worker used local rule-based and local LLM execution to **validate the protocol**. It was a **Reference Execution Implementation**, not an OpenClaw native runtime integration.
>
> **v0.14.1** builds on the same protocol but adds the **OpenClawAgentExecutor**, which calls real OpenClaw agents via `openclaw agent --json` — invoking cloud models (MiniMax-M2.5, Qwen-Plus) with full tool access. This is the true OpenClaw native execution path.

## One-line Value

> Work Orders are now executed by real OpenClaw cloud agents, with full provenance tracking (model, tokens, duration), and graceful fallback to local LLM.

## What Changed

### 1. Executor Abstraction Layer

All executors implement a clean interface:

```
TaskExecutor
├── EchoExecutor          — rule-based, for testing protocol
├── LocalLLMExecutor      — Ollama (fallback)
└── OpenClawAgentExecutor — openclaw agent --json (primary, cloud)
```

**Interface:**
```python
class TaskExecutor:
    def can_handle(task_card) -> bool
    def execute(task_card) -> ExecutionResult
```

**Feature flag:** `OPENCLAW_EXECUTOR_MODE`
- `auto` (default): try OpenClawAgent → fallback LocalLLM → needs_review
- `openclaw_native`: OpenClawAgent only, no fallback
- `local_llm`: Local Ollama only
- `echo`: Rule-based echo only

### 2. OpenClawAgentExecutor — Real Cloud Model Execution

Calls `openclaw agent --agent <name> --message "<prompt>" --json`

Key features:
- **Agent mapping**: task_type → OpenClaw agent (research→research-agent, customer→content-manager, finance→finance-analyst, etc.)
- **Task card override**: `executor_config.openclaw_agent`
- **Timeout enforcement**: 120s default
- **No shell injection**: subprocess with arg array
- **Graceful error handling**: timeout → failed, bad JSON → needs_review, empty output → fallback

### 3. Result Manifest Provenance (v1.2)

Every result.json now includes full execution provenance:

```json
{
  "executor_type": "openclaw_agent",
  "executor_name": "OpenClawAgentExecutor",

  "model_provider": "minimax-cn",
  "model_name": "MiniMax-M2.5",
```

## Architecture (Corrected v0.14.1)

```
CEO / Work Order
  → Skill Router (external_interaction)
  → openclaw_bridge_v2 Dispatch
  → ~/.ai-company-os/openclaw/inbox/WO-xxx.task.json
  → BridgeReferenceWorker
      → Claim (inbox → working/)
      ↓
  Executor Interface <── OPENCLAW_EXECUTOR_MODE
  ├── OpenClawAgentExecutor (primary) ← openclaw agent --json
  │     └── Real cloud model: MiniMax-M2.5 (minimax-cn)
  │     └── Full tool access: Feishu, GitHub, Tavily, browser
  │     └── Full provenance: tokens, model, duration, runId
  ├── LocalLLMExecutor (fallback) ← Ollama (deepseek-r1:8b)
  └── EchoExecutor (fast path)
      ↓
  result.json (Manifest v1.2 with provenance)
  → Callback API → Work Order backfill
  → CEO Summary (🔧[OpenClaw] badge)
```

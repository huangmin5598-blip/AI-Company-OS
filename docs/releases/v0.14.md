## One-line Value

> OpenClaw is now a real employee — reads inbox tasks, executes them, and writes results back.

## What Changed

### 1. OpenClaw Inbox Worker (`bin/openclaw_worker.py`)
A standalone CLI script that:
- Watches `~/.ai-company-os/openclaw/inbox/` for pending task cards
- Reads `WO-xxx.task.json` and validates schema
- Claims tasks (moves to `working/`)
- Executes via TaskExecutor
- Writes `result.json` + output artifacts
- Optionally calls the backend Callback API

Usage:
```bash
python3 bin/openclaw_worker.py          # Process 1 task
python3 bin/openclaw_worker.py --all    # Process all pending
python3 bin/openclaw_worker.py --watch  # Keep watching
python3 bin/openclaw_worker.py --call-backend  # + callback API
```

### 2. Task Executor (`backend/app/services/openclaw_worker/executor.py`)
Supports 3 safe task types:

| Task Type | Description | LLM | Confidence |
|-----------|-------------|-----|------------|
| `echo_test` | Echo input context back | No | 1.0 |
| `read_context_and_write_summary` | Generate structured summary | Yes (Ollama) | 0.82 |
| `file_analysis` | Read + analyze safe files | Yes (Ollama) | 0.82 |

Graceful degradation: when LLM is unavailable, falls back to template-based summary.

### 3. Result Manifest v1.1
Enhanced result.json with:
- `steps`: step-by-step execution trace
- `started_at` / `finished_at`: execution timestamps
- `executor`: "openclaw-worker-lite"
- `errors`: error collection
- `confidence`: 0.0–1.0 quality score

### 4. Skill Router Updates
Added 3 new task_type → capability mappings:
- `echo_test` → `external_interaction`
- `read_context_and_write_summary` → `external_interaction`
- `file_analysis` → `external_interaction`

## Architecture

```plaintext
CEO / Work Order
  → Skill Router (external_interaction)
  → openclaw_bridge_v2 Dispatch
  → ~/.ai-company-os/openclaw/inbox/WO-xxx.task.json
  → OpenClaw Worker (bin/openclaw_worker.py)
      → Claim (move to working/)
      → Task Executor
          ├── echo_test: input → echo_output.txt
          └── read_context_and_write_summary: input → LLM → summary.md
      → result.json (Manifest v1.1)
      → Callback API → Work Order backfill
  → CEO Summary (🔧[OpenClaw] badge)
```

## How to Run the Worker

```bash

# With callback to backend:
cd backend/ && python3 ../bin/openclaw_worker.py --call-backend

# Watch mode (poll every 30s):
cd backend/ && python3 ../bin/openclaw_worker.py --watch

# E2E verification:
```

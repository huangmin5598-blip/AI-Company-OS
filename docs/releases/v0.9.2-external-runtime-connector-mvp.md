## 版本目标

v0.9.2 将 AI Company OS 从"仅支持本地 Codex"扩展为**统一 Runtime 接入框架**，支持本地 Agent、实验性 Agent、云端 Agent 预留的注册与选择。

**核心价值：**
- AI Company OS 不再绑定某个特定 Agent
- 可以为不同任务选择不同 Runtime
- 为后续跨云 Agent 接入奠定安全底座

---

## 交付能力

### Sprint 1 — External HTTP Agent Adapter（通用远程 Agent 模板）

| 能力 | 说明 |
|:----|:------|
| RuntimeAdapter 协议 | 6 个标准方法：health_check / get_capabilities / get_cost / execute / create_session / cancel_session |
| 端点白名单校验 | 仅允许 `http://localhost` / `http://127.0.0.1` / `https://`，禁止 file/ftp/metadata IP |
| Auth 安全 | config 仅存环境变量名，值从 `os.environ.get()` 读取；无 env var 时健康检查跳过 |
| execute() 硬禁用 | 即使配置了 execute_path，也返回 `{"status": "unsupported"}` |
| 禁用 runtime 零触达 | `enabled=0` 时跳过所有请求 |

**关键文件：** `backend/app/runtime/adapters/external_http_adapter.py`

### Sprint 2 — Claude Code Experimental Spike

| 能力 | 说明 |
|:----|:------|
| 版本升级 | v2.1.47 → v2.1.150（brew 路径冲突修复） |
| health_check | ✅ 真实调用 `claude -p 'health check'`，返回实时状态 |
| capability 发现 | ✅ 返回硬编码能力列表（generate_plan, generate_patch, apply, rollback） |
| generate_plan | ✅ **25 秒**返回完整 plan_summary（非交互模式） |
| generate_patch/apply | ❌ 被 experimental 安全门禁拒绝 → 422 |
| ACP 模式 | ❌ 非 TTY 下 stdin 阻塞，标记 experimental_unavailable |

**关键文件：** `backend/app/runtime/adapters/claude_code_stub.py`, `backend/app/runtime/code_capable/claude_adapter.py`

### Sprint 3 — Runtime 端点配置（3 个 disabled cloud runtime）

3 个云端 Runtime 作为 reserved slot 注册，`enabled=0`，不触达远程：

| runtime_id | 类型 | 状态 |
|:-----------|:----|:----:|
| cloud-openclaw | external | 🔌 disabled |
| cloud-hermes | external | 🔌 disabled |
| minimax-agent | external | 🔌 disabled |

**关键文件：** `backend/db/seed_runtimes.py`

### Sprint 4 — CCR runtime_id 参数 + 安全边界

所有 Code Change Request 端点新增 `runtime_id` 参数：

- POST create-ccr
- POST generate-plan
- POST approve-plan
- POST execute-plan

安全边界验证：

| 条件 | 行为 |
|:----|:-----|
| runtime_id 存在且 enabled | ✅ 正常调度 |
| runtime_id 不存在 | ❌ 422 |
| runtime_id 存在但 disabled | ❌ 422 |
| runtime_id 省略 | ✅ 默认 codex |
| claude-code + patch/generate_patch/apply/rollback | ❌ 422 (experimental) |

**关键文件：** `backend/app/routers/code_change_requests.py`

## 技术债务

- Claude Code ACP 模式在非 TTY 环境不可用（stdin 阻塞）
- 3 个 cloud runtime 仅有配置，无真实连接
- execute() 硬禁用，真实远程执行需 v1.x
- 无前端 Runtime 选择 UI（计划 v0.9.4+）
- 无 Auto 选择逻辑（计划 v1.x）

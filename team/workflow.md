# Workflow

## Daily Flow

### Morning (09:00)
1. **Heartbeat** — Check for new tasks
2. **Scan Projects** — What's due today?
3. **Generate Tasks** — Create task cards
4. **Dispatch** — Assign to execution agents

### Throughout the Day
1. **Execute** — Agents work on tasks
2. **Review** — Project leads validate
3. **Progress** — Move tasks forward
4. **Export** — Save deliverables

### Evening (18:00)
1. **Daily Report** — What was done?
2. **KPI Check** — Production metrics
3. **Bottleneck Check** — What's stuck?

---

## Project Flow

```
Opportunity → Task Breakdown → Execution → Review → Deliverable → Monetization
```

---

## Types of Work

| Type | Frequency | Example |
|------|-----------|---------|
| Daily | Every day | novel-v1 (2 posts/day) |
| Weekly | Every week | Research (3 opportunities) |
| One-time | As needed | Build a new feature |

---

## Quality Control

Every task must pass:
1. **Review** — Did we build what was asked?
2. **Validation** — Does it work?
3. **Export** — Is it saved properly?

---

## Failure Handling

If something breaks:
1. **Detect** — System notices
2. **Recover** — Auto-retry or fix
3. **Log** — Record what happened
4. **Report** — Tell the Founder

---

## What We Don't Do

- ❌ Random experiments
- ❌ Endless discussions
- ❌ Untracked tasks
- ❌ Jobs without review

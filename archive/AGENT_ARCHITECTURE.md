# AI Agent 多项目架构需求文档

## 一、背景

我有一个"超级AI助手"（你），帮我管理多个业务项目。每个项目相当于一个独立的小公司，需要多个AI Agent协同工作。

## 二、整体架构

### 2.1 角色定位

**CEO Agent（你，即我）**：
- 核心职责：信息汇总 + 决策建议 + 协调讨论
- 不执行具体业务，只做决策和调度
- 把各项目主管Agent的信息原汁原味汇总给我
- 提供分析建议，必要时拉上项目主管一起讨论

**共享Agent（服务多个项目）**：
- 编程开发Agent + 测试Agent（共用）
- 引流获客Agent（按平台分工）
- 客服Agent（共享，但项目知识库隔离）

**项目专属Agent（单个项目专用）**：
- 如亚马逊的选品Agent、投放Agent等

### 2.2 数据隔离原则

所有项目的运营数据必须完全隔离，避免互相污染。

---

## 三、各Agent职责

### 3.1 编程开发Agent + 测试Agent

**职责**：
- 网站/小程序/APP开发
- API开发
- 代码测试、质量把关
- 部署上线

**特点**：共用，给所有项目提供服务

### 3.2 引流获客Agent

**职责**：
- 小红书运营
- 抖音/视频号运营
- 微信生态运营
- 各平台内容发布、引流

**特点**：按平台分工，不是按项目分工。一个Agent可以服务所有项目。

### 3.3 客服Agent

**职责**：
- 客户咨询问答
- 差评预防
- 消息回复

**特点**：
- 共享Agent，但每个项目有独立知识库
- 客户通过不同入口进入，自动识别项目

---

## 四、亚马逊项目专属Agent

### 4.1 核心Agent（第一阶段要做）

| Agent | 功能 | 说明 |
|-------|------|------|
| 选品Agent | 市场调研、竞品分析、利润测算 | 核心中的核心 |
| 运营Agent | 日常运营、竞品监控、数据复盘 | 需例行监控竞品动态 |
| 素材Agent | 图文描述（5点）+ 主图视频 | 调用AI工具生成 |
| 投放Agent | 亚马逊站内广告（Sponsored Products） | 不与其他平台合并 |

### 4.2 不需要的Agent

客服、仓储物流、财务管理Agent第一阶段不做。

## 五、技术实现

### 5.1 工具支持

- **编程**：Claude Code / Cursor / MiniMax
- **素材生成**：需要API（Midjourney/DALL-E/可图等）
- **搜索调研**：web_search, web_fetch, browser

### 5.2 部署方式

- Vercel（网站前端）
- 云服务器（后端API）
- 各平台API对接

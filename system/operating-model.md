# Operating Model

## Operating Logic

AI Company Model operates as a layered collaboration structure.

The system works through four core layers:

1. **Founder**  
   Defines direction, priorities, and final decisions.

2. **CEO Agent**  
   Translates intent into structured work, coordinates execution across the system, and oversees progress.

3. **Project Leads**  
   Own project-level planning, task decomposition, validation, and delivery structure.

4. **Shared Capability Agents**  
   Provide execution support, implementation support, and creative or content support across projects when needed.

## Collaboration Flow

A typical operating cycle follows this pattern:

Founder → CEO Agent → Project Lead → Supporting Agents → Project Lead review → CEO Agent summary → Founder review

This flow may vary by project, but the general logic remains the same:
- the Founder sets intent
- the CEO Agent coordinates
- the Project Lead manages project execution
- supporting agents help produce outputs
- results are reviewed, summarized, and recorded

## Role Relationships

| Role | Primary Function | Works With | Notes |
|---|---|---|---|
| Founder | Strategic direction and final judgment | CEO Agent | Human decision-maker |
| CEO Agent | System coordination and oversight | Founder, Project Leads, Supporting Agents | Responsible for experiment-level synthesis |
| Project Leads | Project planning and validation | CEO Agent, Supporting Agents | Own project-specific execution structure |
| Shared Capability Agents | Execution and support capabilities | CEO Agent, Project Leads | Activated as needed by project context |

## How Experiments Become Records

A real system run becomes part of the record through structured documentation.

The typical record formation process is:

1. **Task Initiation**  
   A task or project direction is introduced by the Founder.

2. **Task Structuring**  
   The CEO Agent and Project Lead define scope, responsibilities, and task cards.

3. **Execution**  
   Project work is carried out with support from relevant agents.

4. **Validation**  
   Outputs are reviewed at the project level and then at the Founder level when needed.

5. **Recording**  
   The experiment is documented through:
   - experiment summary
   - project record
   - task cards
   - deliverables record
   - build log
   - screenshots or diagrams where applicable

## What Counts as Evidence

Evidence in AI Company Model must be:

- **factual** — it records what actually happened, not just what was intended
- **verifiable** — it can be checked against outputs, files, screenshots, or structured records
- **structured** — it is organized in a consistent experiment format
- **complete enough** — it includes meaningful outcomes, not only successes

Evidence may include:
- task cards
- deliverables
- build logs
- screenshots
- diagrams
- structured project records
- experiment summaries

Not everything produced during execution automatically becomes evidence.

Only materials that are relevant, reviewable, and appropriate for the public evidence layer should be included in this repository.

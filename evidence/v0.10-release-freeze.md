## Release Summary

AI Company OS v0.10 introduces the **Work Delegation Layer** — the core organizational mechanism that transforms the OS from a single-agent tool into a multi-agent company operating system.

### Core Capability

```
Founder Goal → CEO Decomposition → Skill Router → Work Order → Execution → Evidence
```

For the first time, the OS can:
1. Accept a high-level Founder goal
2. Decompose it into structured Goal Sessions with measurable outcomes
3. Route work orders to the right Skill via deterministic Skill Router
4. Execute across multiple runtimes (local, Code Bridge, OpenClaw)
5. Collect evidence and produce a CEO summary

### Known Issues

- Deployment assistant generates checklist only — actual deploy is manual (by design for v0.10)
- Code Bridge mode requires manual approval — no auto-apply
- OpenClaw Bridge creates task cards but doesn't yet receive callbacks
- No UI for Work Order management (CLI/API only)
- Founder → CEO → Work Order pipeline is Hermes-executed, not fully automated in a loop

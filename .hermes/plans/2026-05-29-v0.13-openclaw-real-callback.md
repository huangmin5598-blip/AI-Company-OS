# v0.13 OpenClaw Bridge Real Callback MVP — Implementation Plan

> **For Hermes:** Execute plan step-by-step using direct file writes + terminal verification.

**Goal:** Transform OpenClaw from a task-card-producer into an execution runtime with Inbox/Outbox contract, result manifest backfill, Callback API, and CEO summary integration.

## Task 4: Callback API Endpoint — POST /openclaw-callback

**Objective:** Add `POST /api/v1/work-orders/{work_order_id}/openclaw-callback` with API key auth + idempotency.

**Files:**
- Modify: `backend/app/routers/work_orders.py`
- Modify: `backend/app/config.py`
- Create: `backend/app/services/openclaw_callback.py`

### Step 1: Create openclaw_callback.py service

Service handles:
- API key validation (compare against configured key)
- Callback body validation (required fields: status, result_summary)
- Idempotency check (if WO already completed, reject unless force=true)
- State transition validation (completed → completed = idempotent, failed → completed = OK)
- Backfill: update WO output_path, result_summary, artifacts_json, completed_at
- Update execution_log_json with callback event

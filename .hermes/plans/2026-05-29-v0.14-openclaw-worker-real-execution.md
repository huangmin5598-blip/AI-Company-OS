# v0.14 OpenClaw Worker Real Execution MVP — Implementation Plan

> **Goal:** Turn OpenClaw from "can receive work orders" into "can actually execute a low-risk task and backfill results".

**Architecture:** A standalone Python worker script watches `~/.ai-company-os/openclaw/inbox/`, claims tasks, executes them (echo_test or read_context_and_write_summary using local LLM), writes result.json + artifacts, and optionally calls the callback API.

**Tech Stack:** Python, local LLM (Ollama/Hermes via HTTP API), JSON file protocol

**Execution:** Use `write_file()` direct writes (most reliable on this machine).

## Task 2: OpenClaw Inbox Worker (standalone CLI script)

**Objective:** Build `openclaw_worker.py` — a one-shot CLI script that:
1. Scans inbox/ for pending tasks
2. Reads the first unclaimed task
3. Claims it (move to working/)
4. Executes via TaskExecutor
5. Writes result.json + artifacts
6. Optionally calls callback API

**Files:**
- Create: `bin/openclaw_worker.py` (standalone, no FastAPI deps)
- Create: `backend/app/services/openclaw_worker/worker.py` (shared worker logic)

### Step 1: Create the worker script

Supports flags:
- `--once` (default): process one task and exit
- `--watch`: keep running, poll every N seconds
- `--poll-interval N`: override polling interval
- `--call-backend`: call callback API after completion

### Step 2: Test manually

```
python3 bin/openclaw_worker.py --once
```
